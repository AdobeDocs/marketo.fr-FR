# Configuration de l’accès push iOS pour l’application mobile {#configure-mobile-app-ios-push-access}

1. Cliquez sur **Administration**.

   ![](assets/image2015-4-22-16-3a12-3a32.png)

1. Sélectionner **Applications mobiles**.

   ![](assets/image2015-4-22-16-3a14-3a29.png)

1. Sélectionnez l’application mobile souhaitée.

   ![](assets/image2015-4-22-16-3a33-3a19.png)

1. Sous Type d’accès push, sélectionnez iOS, puis cliquez sur **Configurer**.

   ![](assets/image2016-6-10-11-3a37-3a9.png)

   >[!NOTE]
   >
   >Vous aurez besoin d’une **Certificat** et **Mot de passe** du développeur de vos applications mobiles. Le développeur les reçoit en se connectant au Centre des développeurs Apple, en configurant et en téléchargeant un certificat de notification push pour votre application, puis en exportant le contenu. Le développeur définit le mot de passe lors de l’exportation. **IMPORTANT**: Le certificat doit être adapté au type d’environnement que vous utilisez — Environnement de test ou Production. Vérifiez cette information auprès de votre administrateur Marketo ou de votre développeur d’applications mobiles.

1. Sélectionnez votre certificat, saisissez votre mot de passe, puis cliquez sur **Enregistrer**.

   ![](assets/image2015-4-22-17-3a19-3a18.png)

Bon boulot! Veillez également à configurer l’application avec Android.
